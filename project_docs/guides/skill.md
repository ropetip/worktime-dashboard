# 프로젝트 스킬 노트 (Skill Note)

## Vite 프로젝트 실행 및 빌드 명령어
Vite를 사용하는 프로젝트에서는 목적에 따라 다른 스크립트를 실행해야 합니다.

1. **`npm run dev`**: 개발용 서버 실행
   - 실시간 코드 변경이 반영되는 개발 서버를 시작합니다.
   - 기본적으로 `localhost:5173`에서 실행됩니다.

2. **`npm run build`**: 프로덕션 빌드
   - 프로젝트를 배포하기 위해 브라우저가 이해할 수 있는 정적 파일(HTML, CSS, JS)로 변환하고 압축합니다.
   - 결과물은 `dist` 폴더에 저장됩니다. **이 명령어는 서버를 실행하지 않습니다.**

3. **`npm run preview`**: 빌드 결과물 미리보기
   - `npm run build`로 생성된 `dist` 폴더의 결과물을 로컬 웹 서버로 띄워 실제 배포 환경과 비슷하게 테스트해 볼 수 있습니다.

## 서버 실행 문제 해결 (Q&A)

### Q1. CMD 창을 끄면 서버가 같이 종료돼요.
A. `npm run dev`는 터미널(CMD)의 종속 프로세스로 실행됩니다. 따라서 명령어를 입력한 창을 닫으면 실행 중인 서버 프로세스도 함께 종료되는 것이 정상입니다.

### Q2. 창을 닫아도 서버를 계속 띄워둘 수 없나요?
개발 단계에서는 터미널을 열어두는 것이 원칙(로그 확인 및 코드 수정 반영)이지만, 창이 거슬린다면 다음과 같은 대안이 있습니다.

1.  **VS Code 통합 터미널 사용**: 로컬에 VS Code가 설치되어 있다면, 별도의 CMD 창 대신 VS Code 하단의 터미널(`Ctrl + ` `)에서 실행하세요. 에디터를 닫지 않는 한 유지됩니다.
2.  **백그라운드 실행 (강제)**: 윈도우에서는 `start /B npm run dev`와 같은 명령어를 사용하거나, `pm2` 같은 프로세스 관리 도구를 설치하여 백그라운드에서 실행할 수 있습니다. (다만, 개발 중에는 권장하지 않습니다.)
3.  **여러 개의 CMD 활용**: 하나는 서버를 띄워두고, 작업용 CMD는 별도로 띄워서 사용하세요.
