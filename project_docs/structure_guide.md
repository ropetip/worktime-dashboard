# 📅 파트 근무 현황 대시보드 프로젝트 가이드

이 문서는 개발 지식이 없더라도 본 프로젝트의 구조와 각 파일이 어떤 역할을 하는지 쉽게 이해할 수 있도록 설명합니다.

## 🏗️ 전체 구조 (어떻게 돌아가나요?)

본 프로젝트는 **React(리액트)**라는 도구를 사용하여 만들어진 현대적인 웹 애플리케이션입니다. 
전체적인 작동 원리는 다음과 같습니다.

1.  **데이터 보관**: 여러분이 입력한 근무 일정은 브라우저(크롬, 엣지 등)의 임시 저장소(`localStorage`)에 보관됩니다. (추후 Supabase 연동 시 안전한 서버 DB에 저장됩니다.)
2.  **화면 구성**: 프로그램이 실행되면 캘린더와 대시보드라는 조각(컴포넌트)들이 합쳐져 하나의 화면을 만듭니다.
3.  **자동 계산**: 08시 출근 로테이션과 18시 상주 인원 부족 알림은 프로그램 내부 로직에 의해 실시간으로 계산됩니다.

## 📂 주요 파일 및 폴더 설명

### 1. `src/lib/` (핵심 로직 - "브레인")
-   **`rotationLogic.js`**: 이 프로젝트의 가장 중요한 부분입니다. 준, 지원, 미진 등 파트원들의 출근 순서를 정하고, 대현님의 화요일 특수 규칙 등을 계산하는 규칙이 들어있습니다.
-   **`supabase.js`**: 나중에 데이터를 인터넷 서버에 저장할 때 필요한 연결 통로 역할을 합니다.

### 2. `src/components/` (화면 조각 - "부품")
-   **`Calendar.jsx`**: 큰 달력을 화면에 그려줍니다. 날짜를 클릭하거나 다음 달로 넘어가는 기능을 담당합니다.
-   **`Dashboard.jsx`**: 화면 상단의 통계 판입니다. 오늘 몇 명이 몇 시에 출근하는지, 18시 인원이 충분한지 숫자로 보여줍니다.
-   **`ScheduleModal.jsx`**: "일정 추가" 버튼을 눌렀을 때 나타나는 팝업창입니다. 정보를 입력받는 역할을 합니다.
-   **`ShiftTag.jsx`**: 달력 칸 안에 들어가는 작은 이름표(예: "준 08:00") 하나하나를 만듭니다.

### 3. `src/styles/` (디자인 - "옷")
-   **`globals.css`**: 글자 크기, 색상, 배치, 버튼 모양 등 모든 시각적인 디자인 규칙이 여기에 정의되어 있습니다.

### 4. 기타 중요 파일
-   **`App.jsx`**: 모든 부품(`components`)들을 한데 모아 조립하는 메인 설계도입니다.
-   **`index.html`**: 웹 브라우저가 가장 먼저 읽는 파일로, 전체 프로그램이 시작되는 입구입니다.
-   **`package.json`**: 이 프로그램을 실행하기 위해 필요한 도구들의 명단이 적혀 있습니다.

## 🚀 실행 및 관리 방법

1.  **보기**: `npm run dev` 명령어로 로컬에서 테스트 화면을 띄울 수 있습니다.
2.  **배포**: `npm run build` 명령어를 실행하면 `dist`라는 폴더가 생깁니다. 이 폴더 안에 있는 파일들을 GitHub 등에 올리면 다른 사람들도 볼 수 있는 주소가 생깁니다.
