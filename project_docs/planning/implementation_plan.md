# 프로젝트 구현 계획: 근무 현황 대시보드 마이그레이션

이 계획은 기존 JSP 기반의 단일 파일 시스템을 현대적인 정적 웹 애플리케이션(SPA)으로 전환하고, 향후 인증 및 데이터베이스 연동이 가능하도록 구조를 개선하는 것을 목표로 합니다.

## 사용자 검토 필요 사항

> [!IMPORTANT]
> **기술 스택 확정**: **React + Vite** 환경으로 프로젝트를 구성합니다. GitHub Pages와 Supabase를 결합하여 서버리스 아키텍처를 구현합니다.
> 
> **데이터베이스 및 인증**: **Supabase**(PostgreSQL + Auth)를 사용하여 실시간 데이터 동기화 및 사용자 인증(인가) 기능을 도입합니다.

## 제안된 변경 사항

### 1. 프로젝트 구조 및 컴포넌트 설계
React의 선언적 UI를 활용하여 기능을 독립적인 컴포넌트로 분리합니다.

#### [NEW] `src/components/`
- **`Dashboard.jsx`**: 시간대별 출근 인원 통계 및 18시 상주 인원 검증 로직 포함
- **`Calendar.jsx`**: 월간 캘린더 그리드, 날짜 계산 및 휴일 처리
- **`ScheduleModal.jsx`**: 일정 추가/수정/삭제를 위한 폼 인터페이스
- **`ShiftTag.jsx`**: 캘린더 내 개별 근무 정보 표시 (색상 및 애니메이션 적용)

#### [NEW] `src/lib/`
- **`supabase.js`**: Supabase 클라이언트 초기화 및 DB 연동 함수
- **`rotationLogic.js`**: 준, 지원, 미진, 주봉, 대현 5인의 08시 출근 로테이션 알고리즘

### 2. 데이터베이스 및 인증 (Supabase)
- **Tables**: `members` (파트원 정보), `schedules` (일일 근무 정보)
- **Auth**: 향후 GitHub 로그인 또는 이메일 로그인을 통한 인증/인가 구현
- **Realtime**: 실시간 데이터 구독을 통해 다른 사용자의 변경사항 즉시 반영

### 3. 기능 마이그레이션
- **로테이션 알고리즘**: 기존의 JS 기반 로직을 독립된 모듈로 추출하여 테스트 가능하게 만듦
- **대시보드**: 18시 상주 인원 자동 계산 및 경고 시스템 가독성 개선
- **인터렉션**: 모달 반응성 및 애니메이션 강화 (Glassmorphism 적용)

## 검증 계획

### 자동화 테스트
- 로테이션 알고리즘 유효성 테스트 (Jest 등 활용)
- 상주 인원 계산 로직 경계값 테스트

### 수동 검증
- 다양한 화면 크기(반응형)에서 캘린더 레이아웃 확인
- 일정 추가/수정/삭제 시 실시간 통계 반영 확인
- GitHub Pages 배포 후 접근성 확인
