# 프로젝트 핵심 기술 및 로직 정의 (skill.md)

이 문서는 근무 현황 대시보드 시스템의 핵심 비즈니스 로직과 기술적 구현 방법을 정의합니다.

## 1. 근무 로테이션 알고리즘 (Shift Rotation)

시스템의 가장 핵심적인 로직으로, 형평성 있는 근무 배정을 위한 규칙을 정의합니다.

- **대상 멤버**: 준, 지원, 미진, 주봉, 대현 (5명)
- **08:00 출근 배정**:
  - 로테이션 멤버 5명이 순차적으로 배정됨.
  - 주말 및 공휴일은 건너뜀.
- **특이 사항 (대현)**:
  - 매주 화요일은 10:00 출근으로 고정함.
  - 화요일 로테이션 순번이 대현인 경우, 다음 순번으로 미루거나 조정함.
- **기본 근무**: 나머지 멤버는 09:00 출근을 기본으로 함.

## 2. 캘린더 렌더링 엔진

날짜 데이터를 가각 시각화하는 로직입니다.

- **그리드 구조**: 7열(일~토) 레이아웃.
- **데이터 매핑**: `YYYY-MM-DD` 키를 기반으로 일일 근무자 정보를 조회하여 렌더링.
- **반응형 대응**: 모바일 환경에서는 텍스트를 간소화하거나 배지만 표시.

## 3. 유효성 검사 및 통계 (Validation & Stats)

- **18시 상주 충족 여부**:
  - 09:00, 09:30, 10:00 출근자의 합이 2명 이상인지 매일 검증.
  - 부족 시 대시보드 및 캘린더에 즉시 경고 표시.
- **통계 자동 합계**: 시간별 출근 인원수를 실시간으로 계산하여 카드 UI에 반영.

## 4. 데이터 영속성 (Persistence)

- **초기 단계**: 브라우저 `localStorage`를 이용한 임시 저장.
- **고도화 단계**: Supabase(PostgreSQL) REST API를 통한 원격 저장 및 동기화.

## 5. UI/UX 디자인 시스템

- **프리미엄 룩**: Glassmorphism, 부드러운 박스 섀도우, 세련된 그라데이션 활용.
- **디자인 토큰**: CSS Variables를 통해 시간대별 고유 색상(Emerald, Blue, Violet 등)을 관리.
